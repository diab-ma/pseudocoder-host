# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in pseudocoder, please report it through
[GitHub Security Advisories](https://github.com/diab-ma/pseudocoder-host/security/advisories/new).

**Do not open a public issue for security vulnerabilities.**

## What to Expect

- Acknowledgment within 48 hours.
- A fix or mitigation plan within 5 days for confirmed vulnerabilities.
- Credit in the release notes (unless you prefer otherwise).

## Scope

This policy covers the `pseudocoder` host binary and its dependencies.
The mobile client is closed-source and not covered by this policy.

## Security Model

Pseudocoder is designed for use on trusted networks (LAN or Tailscale).
See the [README](README.md) for security guidance on deployment.
